# SKILL.md — Email Sequence Design

> Design and write automated email sequences that nurture relationships, drive action, and convert.

---

## Overview

This skill produces complete, ready-to-implement email sequences. Each email includes subject line, preview text, full body copy, CTA, timing, and segmentation notes. The deliverable includes a sequence map overview and a metrics plan.

### Workflow

```
Phase 1 — Gather Context (sequence type, audience, goals)
Phase 2 — Design Sequence Map (emails, timing, flow)
Phase 3 — Write Email Copy (email by email)
Phase 4 — Review & Polish (quality check, consistency)
Phase 5 — Deliver (complete sequence document + metrics plan)
```

---

## Phase 1 — Gather Context

**Goal:** Understand who receives these emails, what triggers the sequence, and what success looks like.

### 1.1 Required Context

| Field | What to Ask | Example |
|-------|-------------|---------|
| **Sequence type** | Welcome / Nurture / Re-engagement / Trial expiration / Onboarding / Post-purchase | Welcome sequence |
| **Trigger** | What event starts this sequence? | "User signs up for free account" |
| **Audience** | Who receives these emails? | "New free-tier users, mostly D2C founders" |
| **Primary goal** | What's the #1 conversion you want? | "Activate → use core feature within 7 days" |
| **Product/offer** | What are you selling? | "E-commerce analytics platform" |
| **Existing emails** | Do you have emails running now? | "Just a welcome email, nothing else" |

### 1.2 Optional Context

| Field | Default |
|-------|---------|
| **Brand voice** | Professional but friendly, conversational |
| **Email tool** | Generic (platform-agnostic) |
| **Other sequences running** | None (standalone) |
| **Known objections** | (Derive from product context) |
| **Proof points** | (Use placeholders) |

## Phase 2 — Design Sequence Map

**Goal:** Define the sequence structure — how many emails, what each one does, and when it sends.

### 2.1 Sequence Type Templates

**Welcome / Onboarding (5-7 emails, 12-14 days)**

| # | Timing | Purpose | Subject Pattern |
|---|--------|---------|-----------------|
| 1 | Immediate | Welcome + deliver promised value | "Welcome! Here's your [thing]" |
| 2 | Day 1-2 | Quick win / first success | "The fastest way to [outcome]" |
| 3 | Day 3-4 | Story / why this exists | "Why we built [product]" |
| 4 | Day 5-6 | Social proof | "How [customer] achieved [result]" |
| 5 | Day 7-8 | Overcome #1 objection | "The #1 question we get" |
| 6 | Day 9-11 | Core feature highlight | "[Feature] just changed everything for [audience]" |
| 7 | Day 12-14 | Conversion push | "[Number] teams use [product] to [outcome]" |

**Lead Nurture (6-8 emails, 2-3 weeks)**

| # | Timing | Purpose |
|---|--------|---------|
| 1 | Immediate | Deliver lead magnet + intro |
| 2 | Day 2-3 | Expand on topic |
| 3 | Day 4-5 | Problem deep-dive |
| 4 | Day 6-8 | Solution framework |
| 5 | Day 9-11 | Case study |
| 6 | Day 12-14 | Differentiation |
| 7 | Day 15-18 | Objection handler |
| 8 | Day 19-21 | Direct offer |

**Trial Expiration (3-4 emails)**

| # | Timing | Purpose |
|---|--------|---------|
| 1 | 3 days before | Value recap + what you'll lose |
| 2 | 1 day before | Urgency + special offer (if applicable) |
| 3 | Day of expiration | Last chance + easy conversion CTA |
| 4 | 3 days after | Win-back + "we miss you" |

**Re-engagement (3-4 emails, 2 weeks)**

| # | Timing | Purpose |
|---|--------|---------|
| 1 | Day 0 | Check-in (genuine concern) |
| 2 | Day 3-5 | Value reminder (what's new) |
| 3 | Day 7-10 | Incentive (special offer) |
| 4 | Day 14 | Breakup email (stay or unsubscribe) |

**Post-Purchase (4-5 emails)**

| # | Timing | Purpose |
|---|--------|---------|
| 1 | Immediate | Order confirmation + what's next |
| 2 | Day 1 | Getting started / setup guide |
| 3 | Day 3-5 | Tips for getting more value |
| 4 | Day 7-10 | Ask for review / feedback |
| 5 | Day 14-21 | Cross-sell / upsell / referral |

### 2.2 Timing Principles

- Welcome email: always immediate
- Early sequence: 1-2 days apart (momentum)
- Mid sequence: 2-4 days apart (breathing room)
- Late sequence: weekly (don't overstay)
- B2B: avoid weekends
- B2C: test weekends

### 2.3 Sequence Map Format

Present to user for approval:

```
Sequence: [Name]
Trigger: [Event]
Length: [X emails over Y days]
Exit conditions: [When they leave the sequence]

Email 1 (Day 0): [Purpose]
Email 2 (Day X): [Purpose]
...
```

## Phase 3 — Write Email Copy

**Goal:** Write complete copy for every email in the sequence.

### 3.1 Email Structure

Every email follows this structure:

```
1. HOOK — First line grabs attention
2. CONTEXT — Why this matters to them right now
3. VALUE — The useful content / insight / story
4. CTA — What to do next (one primary action)
5. SIGN-OFF — Human, warm close
```

### 3.2 Subject Line Rules

| Rule | Detail |
|------|--------|
| Length | 40-60 characters ideal |
| Style | Clear > clever, specific > vague |
| Approach | Benefit-driven or curiosity-driven |
| Personalization | Use [First name] where available |
| Emoji | Test, don't assume (they're polarizing) |

**Subject line formulas:**
- Question: "Still struggling with [X]?"
- How-to: "How to [outcome] in [timeframe]"
- Number: "3 ways to [benefit]"
- Direct: "[First name], your [thing] is ready"
- Story tease: "The mistake I made with [topic]"

### 3.3 Preview Text Rules

- 90-140 characters
- Extends the subject line (doesn't repeat it)
- Completes the thought or adds intrigue

### 3.4 Body Copy Rules

| Rule | Detail |
|------|--------|
| Paragraphs | Short — 1-3 sentences max |
| White space | Between every section |
| Scannable | Bullet points for lists |
| Bold | Sparingly, for emphasis only |
| Mobile | Most read on phone — design for it |
| Tone | Conversational, first-person, active voice |
| Read aloud | If it doesn't sound human, rewrite it |

### 3.5 Length Guidelines

| Email Type | Word Count |
|-----------|-----------|
| Transactional | 50-125 words |
| Educational | 150-300 words |
| Story-driven | 300-500 words |

### 3.6 CTA Guidelines

- One clear primary CTA per email
- Button for primary action, text link for secondary
- Button text: Action + outcome ("Start your dashboard" not "Click here")
- Place CTA above fold AND at bottom

### 3.7 From Name

Person's name > company name (higher open rates). "Sarah from Acme" > "Acme Analytics."

### 3.8 Per-Email Output Format

```
Email [#]: [Name/Purpose]
Send: [Timing from trigger]
From: [Sender name]
Subject: [Subject line]
Preview: [Preview text]

---

[Full email body copy]

---

CTA: [Button text] → [Link destination]
Segment/Conditions: [If applicable]
Notes: [Any implementation notes]
```

## Phase 4 — Review & Polish

**Goal:** Quality-check the entire sequence for consistency and effectiveness.

### 4.1 Sequence-Level Checklist

- [ ] **Logical flow** — Does each email build on the previous one?
- [ ] **No repetition** — Same point not made in multiple emails?
- [ ] **Escalating value** — Each email delivers more than the last?
- [ ] **Consistent voice** — Same tone and personality throughout?
- [ ] **One CTA per email** — No competing actions?
- [ ] **Subject line variety** — Different patterns across the sequence?
- [ ] **Timing makes sense** — Not too frequent, not too sparse?
- [ ] **Exit conditions clear** — When and why someone leaves the sequence?

### 4.2 Per-Email Checklist

- [ ] Subject line < 60 chars, specific, compelling
- [ ] Preview text extends subject, doesn't repeat
- [ ] Hook in first line
- [ ] One primary CTA
- [ ] Mobile-friendly formatting
- [ ] Sounds human when read aloud

## Theme Selection Guide

When generating the HTML output, **pick the theme that best matches the user's topic and audience**. Do not default to the same theme every time.

### Theme Options

| Theme | File Suffix | Visual Style | Best For |
|-------|-------------|--------------|----------|
| **Default** | `-default.html` | iOS-style light — system blue, clean cards, inbox-native feel | SaaS products, consumer apps, professional email marketing |
| **Dark** | `-dark.html` | Cyberpunk terminal — deep black, neon cyan/green, monospace fonts, scan-line overlay | Developer tools, technical docs, automation platforms, CLI-focused content, anything "engineering/hacker" |
| **Bold** | `-bold.html` | Neo-brutalism/Memphis — cream background, thick black borders, offset shadows, Space Mono, yellow+pink accents | Creative work, marketing, social/viral content, startups, anything that needs to feel energetic and memorable |

### Decision Rules

```
User's topic / audience                  → Recommended theme
────────────────────────────────────────────────────────────
Dev tools, APIs, coding, automation      → dark
Corporate SaaS, enterprise, formal       → default
Marketing, social, brand, creative       → bold
Data, analytics, dashboards, technical   → dark  
E-learning, education, professional docs → default
Startups, indie, viral, fun content      → bold
Writing, journalism, editorial content   → default
Music, gaming, culture, subculture       → dark or bold
Personal brand, portfolio, resume        → bold (if expressive) or default (if conservative)
```

### How to Apply

1. **Detect theme from user's context** — their topic, brand tone, or explicit preference
2. **Load the matching template file** (`email-sequence-template-[theme].html`)
3. **Replace all `{VARIABLE}` placeholders** with generated content
4. **Deliver the file** — no build step required

If the user asks for a specific style or you're unsure, default to **default** and offer alternatives:
> "I used the clean default theme. Want to try the dark cyberpunk or bold neo-brutalism version?"
